# 物流串接決策備忘錄：自建 vs 物流統合 API

更新日期：2026-06-26

## 結論

不要從頭自寫三大超商物流串接。MVP 到第一個正式物流版本，建議採用「物流統合 API 優先」：先評估綠界物流 API，再備選 ezShip / PayNow 類服務。原因是我們是小團隊，沒有足夠人力處理三家超商各自合約、門市選擇、取號、列印、貨態、逆物流、客服與個資保存責任。

短期產品仍維持現在策略：先做物流試算、尺寸重量提醒、手動單號與手動貨態；等有真實寄送交易後，再串統合 API。

## 為什麼不建議自建三家超商串接

### 1. 商務合約成本高

7-11、全家、萊爾富各自串接，通常會涉及商務申請、測試、正式開通、門市資料、物流標籤與營運規則。小團隊一開始逐一談，速度慢、維運重。

### 2. 技術與維運不是一次性

物流 API 不是只打一支 API：

- 門市選擇。
- 暫存物流訂單。
- 出貨取號。
- 標籤列印。
- 貨態查詢或 webhook。
- 退貨/逆物流。
- 例外狀態：退回、未取、破損、遺失、門市關轉。

任何一段壞掉都會變成客服問題。

### 3. 個資責任會上升

正式物流串接可能會處理姓名、手機、門市、地址、物流單號。這會提高隱私權政策、資料保存、刪除、存取權限與資安要求。

### 4. 客服責任會被使用者期待

一旦我們幫使用者取號，使用者會自然期待我們能處理：為什麼寄不出去、為什麼未到店、為什麼退回、為什麼破損、能否理賠。這超出目前 MVP 人力。

## 物流統合 API 的優勢

### 綠界物流 API

綠界開發者文件顯示，其物流服務提供超商「全家便利商店 C2C/B2C」、「7-ELEVEN C2C/B2C」、「萊爾富 C2C/B2C」、「OK C2C」，宅配則包含「黑貓宅急便」與「中華郵政」。文件也提到物流選擇頁面與暫存物流訂單機制，可在出貨時才取號建立正式訂單。

適合我們的原因：

- 一次接多家通路。
- 有物流選擇頁，降低自製門市選擇 UI 成本。
- 可走暫存訂單，符合交易條件先建立、出貨時再確認的流程。
- 未來 Phase 2 可加入黑貓/郵政，但現在先關掉。

### ezShip

ezShip 官網說明其自 2005 年起提供店到店配送服務，包含超商取貨、超商取貨付款、店到店、店退店、店到宅、API 服務等。它也提到與 OK、萊爾富、全家等通路合作。

適合我們的原因：

- 早期網拍/社群賣家情境較接近。
- 店到店、店退店、店到宅等功能與二手交易接近。
- 可作為綠界之外的備選。

## 決策

### 現在：不串正式 API

先做：

- 物流方式選擇。
- 運費欄位。
- 包裹長寬高重量。
- 超商店到店提醒。
- 手動物流單號。
- 手動貨態事件。

### 有 30 筆真實寄送交易後：測統合 API sandbox

優先：綠界物流 API。

備選：ezShip、PayNow 類物流服務。

### 有 100 筆以上寄送交易後：才談正式商務條件

要確認：

- 每筆物流成本。
- 是否支援個人賣家 C2C。
- API rate limit。
- 貨態回傳方式。
- 門市選擇頁可否嵌入。
- 退貨與未取流程。
- 個資保存期限。
- 客服責任歸屬。

## 問物流廠商的 15 個問題

1. 是否支援 7-11、全家、萊爾富 C2C 店到店？
2. 個人賣家是否可用，還是必須公司行號？
3. 是否提供 sandbox？
4. 是否提供門市選擇頁？
5. 是否支援暫存物流訂單？
6. 是否可出貨時才取號？
7. 貨態查詢是 API polling 還是 webhook？
8. 是否支援未取退回狀態？
9. 是否支援逆物流？
10. 物流標籤如何列印？
11. 包裹尺寸重量限制如何取得？
12. 禁運品規範是否可由 API 取得？
13. 手續費與物流費如何計算？
14. 個資保存與刪除責任如何切分？
15. 若包裹遺失/破損，客服與理賠責任由誰處理？

## 實作建議

### Sprint A：手動物流事件

- Worker deal schema 增加物流欄位。
- 賣方操作頁可填物流單號。
- 交易頁顯示物流狀態。

### Sprint B：物流規格資料表

- 前端維護 7-11 / 全家 / 萊爾富常用限制與提醒。
- 不宣稱官方即時資料。
- 顯示「實際以物流商公告為準」。

### Sprint C：綠界 sandbox PoC

- 只在測試環境建立暫存物流訂單。
- 不開放正式用戶取號。
- 先確認流程、資料欄位、錯誤處理。

## 風險邊界

在正式簽約與完整測試前，網站不能寫：

- 已整合 7-11 / 全家 / 萊爾富。
- 一鍵寄件。
- 保證物流追蹤正確。
- 保證理賠。
- 官方合作。

只能寫：

- 支援物流條件紀錄。
- 支援運費與包裹規格提醒。
- 支援手動物流單號與貨態紀錄。
- 未來評估統合物流 API。
