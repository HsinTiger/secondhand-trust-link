# 安全主管交付：MVP 市場驗證與資安守門

更新日期：2026-06-26

## 今日主管結論

先讓網站活下去，不做金流、不做託管、不承諾保證交易。MVP 的唯一核心任務是驗證：高單價二手賣家是否願意把「交易條件連結」貼給買方，讓雙方減少私訊來回、降低各說各話。

## 已落地控制

- 對外文案改為流程紀錄工具，避免誤導成收款、託管、撥款或保證安全。
- Worker 新增聚合 metrics API，只回傳數量與分類，不回傳聯絡方式、買賣方 token、原始留言。
- Dashboard 設定 noindex，避免內部 PM 儀表板被搜尋引擎收錄。
- 高風險品類關鍵字直接拒絕建單：票券、禮品卡、點數、遊戲幣、帳號、精品、代儲、金融、投資、虛擬帳戶、門號。
- 前端預覽與建單結果補 HTML escape，降低 XSS 風險。
- 收緊狀態流程：買方不能在未出貨狀態直接確認完成；seller release 動作不開放。
- 增加全域讀取/建單 rate limit、每筆交易事件總量上限、重複事件擋下。
- GitHub Pages workflow 改為只部署 public 目錄，避免把 repo 文件或未來敏感產物整包公開。

## 7 天市場驗證

### 不做

- 不做 USDC/USDT 金流。
- 不做 escrow 託管。
- 不做 KYC、黑名單、AI 仲裁、群主管理後台。
- 不投廣告。
- 不先做大量 SEO 內容矩陣。

### 做

1. Day 1：列 20 位最近 7 天正在賣二手 iPhone、MacBook、相機、鏡頭的賣家；再列 10 位 FB/LINE 群主。
2. Day 2-3：私訊賣家，不叫他們自己研究，直接說「我幫你建第一筆交易條件連結」。
3. Day 4：只在允許工具/自介的社團貼文，標題避開金流，主打「交易前把驗收條件寫清楚」。
4. Day 5-7：追蹤買方是否打開連結、賣方是否願意再用、哪一欄最有幫助。

## 首批客群

1. 高單價二手 3C 賣家：iPhone、MacBook、相機、鏡頭、顯卡。
2. LINE 代購/小賣家：痛點是跑單、不取貨、私訊資訊散落。
3. FB/LINE 社群管理者：痛點是詐騙糾紛與群規難執行。

## 私訊話術

> 我做了一個免費小工具，不收款、不保管資金，只是幫私下二手交易把商品狀況、金額、出貨期限、驗收期限、爭議紀錄寫成一個連結。你現在這筆商品我可以直接幫你建一版，你只要貼給買方，看對方會不會比較安心。願意試一筆嗎？

## 社團貼文角度

- 徵 10 位二手 3C 賣家測試：交易前把驗收條件寫清楚。
- 私下二手交易不要只靠截圖：我做了一個交易條件連結。
- 買家跑單 / 商品不符 / 面交爭議，有沒有人願意試一個流程紀錄工具？

## 追蹤指標

- 私訊 30 人，至少 6 人回覆。
- 至少 3 人用真實商品建立交易連結。
- 至少 1 人真的把連結貼給買方或社群。
- 至少 10 則具體回饋。
- 痛點排序：不出貨、不取貨、商品不符、面交安全、私訊太多。

## 下一個安全關卡

上線收真實陌生流量前，建議啟用 Cloudflare Turnstile：

1. Cloudflare Dashboard 建立 Turnstile site key / secret key。
2. Worker secret 設定 `TURNSTILE_SECRET_KEY`。
3. 將 `wrangler.toml` 的 `TURNSTILE_ENFORCED` 改成 `true`。
4. 前端嵌入 Turnstile widget 後再部署。
