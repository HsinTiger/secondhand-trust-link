# 老闆提問回覆備忘錄：為什麼電商平台越抽越貴，我們怎麼切入

更新日期：2026-06-26

## 一句話結論

傳統電商平台到後期手續費越抽越高，不是單純貪婪，而是因為它們承擔了流量、金流、物流、客服、退貨、仲裁、補貼、廣告系統與合規成本；當平台掌握賣家的流量入口後，就會自然走向抽成與廣告變現。賣貨便類工具手續費較低，是因為它主要強在物流整合，不是真正完整交易仲裁平台，所以爭議處理體驗容易偏弱。

我們不應該一開始挑戰完整電商平台，也不應該一開始做 USDT / USDC 託管。最好的切入點是：做「私下交易流程與證據層」，不碰資金、不搶物流、不做大賣場，用低成本工具先累積信任、交易紀錄與使用回饋。

## 為什麼蝦皮、露天、樂天、MOMO、PChome 類平台後期會越抽越貴

### 1. 平台越大，固定成本越高

它們不只是商品刊登頁，還包含：

- 流量取得與補貼。
- 金流與退款。
- 物流串接與逆物流。
- 客服與爭議處理。
- 反詐騙、資安、風控。
- 發票、稅務、法遵。
- App、搜尋、推薦、廣告系統。

規模越大，詐騙、退貨、客服、金流風險也越高，最後都會轉嫁到賣家費用、成交手續費、廣告費或排名費。

### 2. 平台掌握流量後，賣家被鎖住

早期平台用低費率吸引賣家；後期賣家的客源、評價、搜尋排名、物流習慣都綁在平台內，轉移成本變高。平台就會從「成交抽成」走向「成交抽成 + 廣告曝光 + 活動費 + 金流/物流服務費」。

### 3. 平台必須偏向標準化，不適合長尾私下交易

大型平台適合標準商品、大量訂單、可規模化客服。但二手商品、面交、瑕疵描述、序號、開箱影片、買家不取貨這些長尾狀況，很難用同一套客服規則公平處理。

### 4. 仲裁成本很難便宜

真正的仲裁需要看聊天紀錄、物流證明、開箱影片、商品狀況、付款證明。這件事高度人工、低毛利、容易得罪雙方。平台若要做強仲裁，成本會上升；若不做，App 評論就會變差。

## 為什麼賣貨便類工具有機會但評論容易差

7-11 / 全家賣貨便的優勢是物流入口、取貨便利、低門檻。但弱點是：

- 它更像物流 + 簡單賣場工具，不是完整仲裁系統。
- 爭議時平台很難判斷商品真假、瑕疵責任、買方是否惡意。
- 對賣家來說，跑單、不取貨、退貨爭議仍痛。
- 對買家來說，商品不符、賣家拖延、客服回應慢仍痛。

所以它能解決「寄送與收款流程」，但不一定能解決「交易前條件講清楚」與「爭議時證據結構化」。

## 我們的切入點：不要做新電商平台，先做交易信任層

我們應該避免變成另一個高成本平台。短期定位：

> 二手安心交易不是賣場，不碰金流，不保管資金；它是私下交易的條件確認、證據留存、驗收期限與爭議紀錄工具。

### 我們要補的是平台中間的空白

傳統平台太重、抽成高；賣貨便類工具仲裁弱；FB / LINE 私下交易太亂。

我們補中間：

- 比大型電商輕。
- 比純私訊有紀錄。
- 比賣貨便更重視交易條件與證據。
- 不一開始承擔金流與仲裁責任。

## 關於 USDT / USDC：現在不要當主賣點

老闆提到的疑慮是對的：會用冷熱錢包、USDT、USDC 的人通常已經有 Web3 門檻，至少要懂交易所 KYC、入金、提幣、鏈別、手續費、地址風險。這對一般二手賣家不是低門檻。

因此策略上：

1. 首頁不要主打 USDT / USDC。
2. MVP 不做平台託管或代收。
3. 先允許使用者記錄「約定付款方式」與「付款證明」，而不是由我們處理付款。
4. 等有一批 Web3 熟悉用戶後，再做「鏈上付款證明」或「智能合約 escrow PoC」。
5. 若未來做穩定幣支付，必須先確認 VASP、電子支付、代理收付、洗錢防制與消費爭議責任邊界。

## 我們如何改善現有問題

### 問題 A：平台抽成太高

我們短期不抽成交手續費，先做免費工具。未來收費也不應該走高抽成，而是：

- 賣家進階模板月費。
- 社團管理工具月費。
- 高價交易證據包匯出費。
- 爭議資料整理費。
- API / 白標工具費。

避免走「掌握流量後抽更重」的老路。

### 問題 B：私下交易無保障

我們不承諾保障，但讓雙方更難裝傻：

- 商品狀況先寫清楚。
- 出貨期限先寫清楚。
- 驗收期限先寫清楚。
- 開箱、序號、物流證明有欄位。
- 爭議時間線有紀錄。

### 問題 C：賣貨便仲裁弱

我們不是一開始做仲裁，而是先把仲裁前的資料結構化。未來如果要做仲裁，可以從「證據包」開始，而不是直接判誰贏。

### 問題 D：Web3 門檻高

不要要求使用者懂錢包。短期只說：

- 不收款。
- 不保管資金。
- 不用註冊先試。
- 先建立交易條件連結。

Web3 只放在未來路線，不放在第一屏。

## 產品路線建議

### Phase 1：流程紀錄工具

目標：收集真實交易回饋。

功能：

- 建立交易條件連結。
- 買方/賣方狀態頁。
- 出貨與驗收事件。
- 風險提示。
- 回饋表單。

成功指標：

- 30 天內 30 位賣家願意試用。
- 100 筆交易條件連結。
- 10 位使用者願意再次使用。

### Phase 2：證據包與社群工具

目標：讓群主與賣家覺得有管理價值。

功能：

- 交易證據包匯出。
- 群規模板。
- 高單價交易檢查表。
- 買賣雙方評分不公開，只做風險提示。

可能收費：

- 社團管理者月費。
- 賣家進階模板。
- 證據包匯出。

### Phase 3：付款證明整合

目標：不碰資金，但降低付款爭議。

功能：

- 付款截圖上傳。
- 匯款後五碼/時間紀錄。
- 鏈上交易 hash 紀錄。
- 不代收、不保管。

### Phase 4：Web3 escrow PoC

只面向懂錢包的小眾使用者，不面向一般大眾。

功能：

- 智能合約 escrow 測試。
- 只支援小額、白名單、測試網或低風險 L2。
- 明確標示不是一般消費者保障服務。

## 商業模式建議

不要走高抽成，改走低固定費與工具費：

1. 免費：建立基本交易連結。
2. NT$29-99/月：賣家常用模板、歷史紀錄、證據包。
3. NT$199-499/月：社團管理者工具、群規模板、風險提醒頁。
4. 單次 NT$10-30：高價交易證據包匯出。
5. 未來 B2B：二手社群、代購社群、寄賣店 API。

## 給老闆的回答版本

我們不要一開始跟蝦皮、露天、賣貨便正面競爭。那些平台抽成變高，是因為它們承擔了完整交易平台的流量、物流、金流、客服、法遵與仲裁成本；而賣貨便類工具費用較低，是因為它主要做物流和簡單賣場，仲裁能力自然有限。

我們的機會是在中間層：不做新賣場、不碰金流、不承諾仲裁，而是先做私下交易的「條件確認與證據流程」。讓 FB、LINE、二手 3C、相機、代購這些交易，在成交前先把商品狀況、金額、出貨、驗收、爭議處理寫清楚。

USDT / USDC 不是第一階段主賣點，因為一般賣家要先 KYC、入金、懂錢包和鏈別，門檻太高。第一階段我們只記錄付款方式與付款證明，不處理資金。等找到一批懂 Web3 的高意願用戶，再做鏈上付款證明或 escrow PoC。

短期我們靠零預算冷啟動：手動找近期正在賣二手 3C / 相機的人，幫他建立第一筆交易條件連結，累積真實回饋與案例。長期商業模式不靠高抽成，而是靠賣家模板、社群管理工具、證據包匯出、API 或白標服務收費。
